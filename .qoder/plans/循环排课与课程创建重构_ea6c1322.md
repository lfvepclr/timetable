## Task 2: 周模式改为循环排课模式

将"周模式"扩展为"循环排课"，支持"天循环"和"周循环"两种模式。

### 2.1 更新数据结构
- 修改 `server/migrations/001_init.sql`: `weekly_patterns` 表添加 `cycle_type TEXT DEFAULT 'weekly'` 列 (向后兼容，已有数据默认为 weekly)
- `local-db.js` 无需修改表结构 (JSON 存储，新字段自动兼容)，但 `generateSchedule` 函数需要支持新逻辑

### 2.2 修改排课模板编辑表单 (`pattern-edit`)
- 修改 `miniprogram/pages/teacher/pattern-edit/pattern-edit.js`:
  - `form` 添加 `cycle_type: 'weekly'` 字段 (默认周循环)
  - 添加 `onCycleTypeChange` 方法处理类型切换
  - 添加 `estimatedCount` 计算方法: 根据 cycle_type + valid_from + valid_until 计算预计课节数
  - `valid_until` 默认设为 valid_from + 90 天 (让用户有默认值)
  - 当 cycle_type 为 'daily' 时，隐藏"周几"选择器
  - `save` 方法保存 `cycle_type` 字段

- 修改 `miniprogram/pages/teacher/pattern-edit/pattern-edit.wxml`:
  - 添加"循环类型"选择器 (天循环 / 周循环) — 用两个 chip 按钮切换
  - "周几"选择器仅在"周循环"模式显示
  - 日期范围标签改为"开始日期"(默认今天)和"截止日期"
  - 显示"预计 X 节课"提示文案

### 2.3 更新排课生成逻辑
- 修改 `miniprogram/utils/local-db.js` 的 `generateSchedule` 函数:
  - 读取 `pattern.cycle_type`，默认 'weekly'
  - `cycle_type === 'daily'`: 日期范围内每天生成一节课 (跳过调休工作日)
  - `cycle_type === 'weekly'`: 保持原逻辑 (匹配 day_of_week)

- 修改 `server/routes/schedule.js` 的 `/generate` 路由: 同步支持 `cycle_type` 逻辑
- 修改 `server/routes/fn.js` 的 `_generateSchedule` 函数: 同步支持
